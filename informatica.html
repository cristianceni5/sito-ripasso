0. Sintassi vs. Semantica & Grammatica C

Sintassi (BNF): È la grammatica (la "forma"). L'albero sintattico verifica se la frase è legale nel linguaggio. Si costruisce usando produzioni come <decl>, <expr>, <var>.

Semantica (Valori): È il significato (valore calcolato) di ogni nodo. All'esame è richiesto di annotare l'albero sintattico con i valori reali.

// Esempio di derivazione semantica per: a = b + c
. Γ(<expr1><op2><expr2>) = Γ(<expr1>) <op2> Γ(expr2)

  a     +     b
  |     |     |
<var>   |   <var>
val:5   |   val:10
  |     |     |
<expr> <op2> <expr>
val:5   |   val:10
  |     |     |
   \    |    /
    \   |   /
     <expr>
     val:15

0.5 Regole di Precedenza (Sintesi)

Per le DICHIARAZIONI (<decl>): Priorità (Alta → Bassa): Parentesi <decl>, Suffissi [] e () (sx→dx), Prefissi * (dx→sx). 

int *A[10];     // A -> A[10] (Array) -> *A[10] (Puntatore ad Array)
int (*A)[10];   // A -> *A (Puntatore) -> (*A)[10] (Array di Puntatori)

Per le ESPRESSIONI (<expr>): Priorità: Parentesi ()/Accesso []/./->, Unari (!, *, &, ++, --), Moltiplicativi, Additivi, Relazionali, Logici (&& poi ||), Assegnamento (bassissima).

1. Espressioni e Operatori

Un'espressione è una combinazione di variabili, costanti e operatori. Può avere side-effect (modificare lo stato, es. ++, --, =) o essere pura. 

<expr> ::= <var> | <const> | <var> ++ | ++ <var> | <var> -- | -- <var>
<var> = <expr> | <expr1><op2><expr2> | <op1><expr1>

Operatori Aritmetici & Forza dei Tipi

int a = 10; int b = 4; float c;
c = a/b; // c sarà 2.0 (troncamento intero prima dell'assegnazione)

// Forza dei tipi: int < float < double | short < int < long | unsigned < int

Operatori Relazionali e Logici (Short-Circuit)

In C non esiste un tipo boolean nativo: 1 = Vero, 0 = Falso. Qualsiasi valore ≠0 è vero.

Short-circuit evaluation: L'operatore && (AND) non valuta expr2 se expr1 è falso. L'operatore || (OR) non valuta expr2 se expr1 è vero. Questo è vitale per ottimizzazione ed evitare segmentation fault (es. check puntatori NULL). 

int a = 6; int b = 3; int c;
c = a < b && a++;  // Se a < b è falso (come qui), a++ NON viene eseguito. a rimane 6.
if (ptr != NULL || *(ptr) == 5) { /* previene deref di NULL */ }

Operatori Bit-a-bit

Lavorano direttamente sui bit: & (AND), | (OR), ^ (XOR), << (Shift sx), >> (Shift dx), ~ (NOT).

2. Puntatori e Array

Puntatori

Variabili che memorizzano indirizzi di memoria. Essenziali per passaggio per riferimento e allocazione dinamica.

<declaration> ::= <type><decl>
<decl> ::= <identifier> | * <decl>
<var> ::= <identifier> | (<var>) | *<expr>  // Dereferenziazione
<expr> ::= ... | &<var>                     // Operatore Indirizzo
int x = 50;
int *ptrX = &x; // ptrX contiene l'indirizzo di x
printf("Valore puntato: %d", *ptrX); // Dereferenziazione, stampa 50

// Cast di puntatori (da void* a tipo specifico)
void *ptr; int m;
ptr = &m;
int n = *((int*)ptr); // (int*) rende legale l'operazione, * dereferenzia

Array (Statica vs Dinamica)

Un array è un blocco di memoria contigua. La dichiarazione fissa la dimensione a compile-time (statica). Il nome dell'array è valutato come l'indirizzo del suo primo elemento: A[i] ≡ *(A + i). 

<decl> ::= <identifier> | *<decl> | <decl>[<const>]
<var> ::= ... | <expr1> [<expr2>]
float **Z[3][2]; // Z è matrice 3x2 di doppi puntatori a float (Occupa 3x2x4B = 24 Byte)

3. Istruzioni (Control Flow) e Funzioni

<statement> ::= <expr>; | <statement1><statement2> | {statement}
<statement> ::= ... | if(<expr_guardia>)<statement> [else<statement>]
<statement> ::= ... | for(<expr_int>; <expr_guard>; <expr_inc>)<statement>
<statement> ::= ... | goto <label>; | break; | continue; | return[<expr>];

Funzioni & Variabili Statiche

Il C utilizza funzioni per partizionare il codice (GOTO mascherato con rientro). Le variabili contrassegnate come static all'interno di una funzione non risiedono nello Stack, ma nel segmento Dati: mantengono il valore tra invocazioni successive.

void f(int a) {
    static int count = 0; // Inizializzata solo una volta
    count++;              // Preserva lo stato per contare le chiamate
}

4. Dati Strutturati (Struct) e Typedef

Le struct aggregano tipi eterogenei. L'allocazione non è sempre strettamente contigua (padding/allineamento imposto dal compilatore). typedef è un alias sintattico per migliorare leggibilità e portabilità.

<struct_definition> ::= struct <identifier> {{<type><decl>;}};
typedef struct {
    int x;
    int y;
} Punto;

struct Lista { int valore; int besello; } lista1;
struct Lista *ptrLista = &lista1;

(*ptrLista).valore += 5;    // Dereferenziazione esplicita
ptrLista->besello += 10;    // Zucchero sintattico (equivalente)

5. Strutture Dati: Liste (Sequenziali e Collegate)

A. Lista Sequenziale (Coda Circolare Statica)

Implementazione su array prefissato (buffer). Usa indici Head e Tail (circolari).
Pro: No frammentazione OS, accesso diretto. Contro: Dimensione fissa, spostamento gravoso di elementi in mezzo. 

struct list { int *buffer; int size; int head; int tail; };

boolean suf_consume(struct list *ptr, int *value) {
    if(ptr->head != ptr->tail) {
        ptr->tail = (ptr->tail - 1 + ptr->size) % ptr->size; // Logica circolare
        *value = ptr->buffer[ptr->tail];
        return TRUE;
    }
    return FALSE;
}

B. Lista Collegata (Dinamica tramite Puntatori)

Elementi allocati a runtime sparsi in heap, collegati da puntatori. Ricorsione formale del tipo consentita unicamente grazie all'indirezione tramite puntatore (altrimenti la dimensione a compile-time sarebbe infinita).

struct list {
    int value;
    struct list *next; // Corretto: usa un puntatore (4B o 8B prefissati)
};

void pre_insert(struct list **ptr, int value) {
    struct list *tmp = (struct list *)malloc(sizeof(struct list));
    tmp->value = value;
    tmp->next = *ptr;
    *ptr = tmp;
}

void suf_insert(struct list **ptr, int value) {
    while(*ptr != NULL) { ptr = &((*ptr)->next); } // Scorre fino al termine
    pre_insert(ptr, value);
}

6. Algoritmi di Ordinamento

Selection Sort

Trova iterativamente il massimo/minimo e lo posiziona (Swap).
Complessità: O(n²) sempre. Spazio: O(1) in-place. Stabile: NO. 

void selection_sort(int *V, int N, int **perm) {
    int iter, count, count_of_max;
    *perm = (int*)malloc(sizeof(int)*N);
    for(count = 0; count < N; count++) (*perm)[count]=count;

    for(iter = 0; iter < N-1; iter++) {
        for(count = 1, count_of_max = 0; count < N-iter; count++) {
            if(V[(*perm)[count_of_max]] < V[(*perm)[count]]) {
                count_of_max = count;
            }
        }
        swap(*perm, count, count_of_max);
    }
}

Bubble Sort

Confronta elementi adiacenti, gli elementi più grandi "affiorano" (bubble up).
Complessità: O(n²) medio/pessimo. Caso migliore: O(n) con check di uscita anticipata (early exit flag). Stabile: SÌ. 

void bubble_sort(int *V, int N) {
    int i, j, tmp, swapped;
    for (i = N-1; i > 0; i--) {
        swapped = 0;
        for (j = 0; j < i; j++) {
            if (V[j] > V[j+1]) {
                tmp=V[j]; V[j]=V[j+1]; V[j+1]=tmp;
                swapped = 1;
            }
        }
        if (!swapped) break;
    }
}

Merge Sort

Divide il vettore a metà, ordina ricorsivamente, poi fonde le due metà (merge).
Complessità: O(n·log n) sempre. Spazio: O(n) extra. Stabile: SÌ. 

void merge(int *V, int left, int mid, int right) {
    int n1=mid-left+1, n2=right-mid, L[n1], R[n2], i, j, k;
    for(i=0;i<n1;i++) L[i]=V[left+i];
    for(j=0;j<n2;j++) R[j]=V[mid+1+j];
    for(i=0,j=0,k=left; k<=right; k++) {
        if(i<n1 && (j>=n2 || L[i]<=R[j])) V[k]=L[i++];
        else V[k]=R[j++];
    }
}
void merge_sort(int *V, int l, int r) {
    if(l >= r) return;
    int m = l + (r-l)/2;
    merge_sort(V, l, m);
    merge_sort(V, m+1, r);
    merge(V, l, m, r);
}

Quick Sort

Sceglie un pivot, partiziona (minori a sx, maggiori a dx), recurse sulle due parti.
Media: O(n·log n). Pessimo: O(n²) su array già ordinato con pivot=ultimo. Spazio: O(log n) stack. Stabile: NO. In pratica il più veloce (cache locality). 

int partition(int *V, int lo, int hi) {
    int pivot=V[hi], i=lo-1, tmp;
    for(int j=lo; j<hi; j++) {
        if(V[j]<=pivot) { i++; tmp=V[i]; V[i]=V[j]; V[j]=tmp; }
    }
    tmp=V[i+1]; V[i+1]=V[hi]; V[hi]=tmp;
    return i+1;
}
void quick_sort(int *V, int lo, int hi) {
    if(lo < hi) {
        int p = partition(V, lo, hi);
        quick_sort(V, lo, p-1);
        quick_sort(V, p+1, hi);
    }
}

8. Algoritmi di Ricerca

Ricerca Sequenziale (Lineare)

Scansione elemento per elemento. Funziona su qualsiasi array (anche non ordinato).
Complessità: O(n) nel caso medio e pessimo. O(1) nel caso migliore (primo elemento). 

int linear_search(int *V, int N, int key) {
    for(int i=0; i<N; i++)
        if(V[i] == key) return i;  // Restituisce indice
    return -1;                      // Non trovato
}

Ricerca Binaria (Binary Search)

Prerequisito: array ordinato. Dimezza lo spazio di ricerca ad ogni passo confrontando con l'elemento centrale.
Complessità: O(log n). Con n=1.000.000 bastano al massimo 20 confronti! Non funziona su array non ordinati. 

int binary_search(int *V, int N, int key) {
    int lo=0, hi=N-1, mid;
    while(lo <= hi) {
        mid = lo + (hi-lo)/2; // Evita overflow rispetto a (lo+hi)/2
        if(V[mid] == key) return mid;
        else if(V[mid] < key)  lo = mid+1;
        else                   hi = mid-1;
    }
    return -1;
}

Ricerca a Salti (Jump Search)

Compromesso tra lineare e binaria. Salta di passi di dimensione √n, poi fa ricerca lineare nel blocco. Complessità: O(√n). Utile quando il costo di tornare indietro (backward) è alto. 

int jump_search(int *V, int N, int key) {
    int step = (int)sqrt(N), prev=0, i;
    while(V[step-1] < key) {
        prev = step; step += (int)sqrt(N);
        if(prev >= N) return -1;
    }
    for(i=prev; i<step && i<N; i++)
        if(V[i]==key) return i;
    return -1;
}

9. Complessità e Notazione Big-O

La complessità computazionale misura quante risorse (tempo, spazio) un algoritmo usa in funzione della dimensione dell'input n. Si usa la notazione O grande (Big-O) che esprime il comportamento asintotico nel caso peggiore.

Classi di Complessità Comuni

NotazioneNomeEsempion=1000 (ops approx)

O(log n)LogaritmicaRicerca binaria~10
O(√n)RadiceJump search~32

Regole per calcolare la complessità

  • Istruzioni semplici (assegnamento, confronto, aritmetica): O(1).
  • Ciclo for che esegue n iterazioni con corpo O(1): O(n).
  • Cicli annidati n×n: O(n²).
  • Divide-and-conquer (dimezza ad ogni passo): O(log n) per trovare, O(n log n) con merge.
  • Si tiene il termine dominante: O(n² + n) = O(n²). Si ignorano costanti: O(3n) = O(n).
// Analisi complessità: quante volte si esegue il corpo del while?
int i = N;
while(i > 1) {
    i = i / 2;  // i dimezza ogni volta: N, N/2, N/4 ... 1 → log₂(N) iterazioni → O(log n)
}

10. Ricorsione vs Iterazione

Un problema ricorsivo si risolve richiamando la stessa funzione su un sotto-problema più piccolo, fino a un caso base che viene risolto direttamente. Ogni chiamata ricorsiva occupa un frame nello stack di chiamata: troppa profondità causa stack overflow.

Regole della Ricorsione

  • Caso base: condizione che termina la ricorsione (es. n == 0).
  • Passo ricorsivo: riduce il problema e richiama sé stesso.
  • Convergenza: ogni chiamata deve avvicinarsi al caso base.
// Fattoriale: ricorsivo O(n) spazio stack, iterativo O(1)
int fact_r(int n) {
    if(n == 0) return 1;   // Caso base
    return n * fact_r(n-1); // Passo ricorsivo
}
int fact_i(int n) {
    int res=1;
    for(int i=2; i<=n; i++) res *= i;
    return res;
}

Ricorsione su Liste Collegate

// Stampa ricorsiva di una lista (visita pre-order)
void print_list(struct list *ptr) {
    if(ptr == NULL) return; // Caso base: lista vuota
    printf("%d ", ptr->value);
    print_list(ptr->next);  // Passo ricorsivo sul resto
}
// Stampa inversa (post-order): prima recurse, poi stampa
void print_reverse(struct list *ptr) {
    if(ptr == NULL) return;
    print_reverse(ptr->next);
    printf("%d ", ptr->value);
}

11. Codifica Numerica e Rappresentazione in Memoria

Complemento a Due (Integers con Segno)

Il C (e MIPS) usa il complemento a due per rappresentare interi con segno. Con n bit si rappresentano valori da -2^(n-1) a +2^(n-1) - 1. Il bit più significativo (MSB) è il bit di segno: 0 = positivo, 1 = negativo. 

// Con 4 bit: range -8 a +7
// +5  = 0101
// -5  = NOT(0101) + 1 = 1010 + 1 = 1011
// -1  = 1111   (tutti uno)
// -8  = 1000   (solo il bit di segno)
// Overflow: +7 + 1 = 0111 + 0001 = 1000 = -8 (wrapping)

IEEE 754 Floating Point (float da 32 bit)

Un float è suddiviso in: 1 bit segno | 8 bit esponente (bias 127) | 23 bit mantissa. Valore = (-1)^s × 1.mantissa × 2^(exp-127).

// -6.5 in IEEE 754:
// -6.5 = -110.1b = -1.101 × 2^2

7. Traduzione C in Assembler MIPS (Architettura R4000)

Il processore esegue istruzioni macchina. L'assembler è la forma simbolica. Le variabili in MIPS sono mappate direttamente nei registri (es. $s0, $t1).

Tipo Operazione Sintassi MIPS e Semantica
Aritmetica add $s0, $s1, $s2 (a = b + c)
addi $s0, $s1, 15 (a = b + 15)
Memoria (Load/Store) lw $s0, 100($0) (Carica word in $s0)
sw $s0, 200($0) (Salva word da $s0)
Flusso (Branch/Jump) bne $s1, $s2, label (If Not Equal)
j label (Jump)
slt $t0, $s1, $t5 (Set on Less Than)

Esempio Parziale: Traduzione Ciclo FOR Array

// C Code Equivalente:
// for (i=1, j=0; i<10; i++) {
//    if (V[i] == W[j]) W[j+1]++; else j += 2;
//    W[j] = V[i]; W[j+1] = 1;
// }

addi $s0, $0, 1      # i = 1
addi $s1, $0, 0      # j = 0
addi $s2, $0, 500    # Base addr V
addi $s3, $0, 540    # Base addr W
addi $t0, $0, 10     # Limite ciclo

start_loop:
    slt  $t3, $s0, $t0   # $t3 = 1 se i < 10
    beq  $t3, $0, fine_loop  # Esci se i >= 10

    sll  $t4, $s0, 2     # i * 4 (offset Byte per int)
    addu $t5, $s2, $t4   # &V[i]
    lw   $t6, 0($t5)     # $t6 = V[i]

    sll  $t4, $s1, 2     # j * 4
    addu $t5, $s3, $t4   # &W[j]
    lw   $t7, 0($t5)     # $t7 = W[j]

    bne  $t6, $t7, ramo_else

    # ... (operazioni ramo IF / ELSE e chiusura ciclo)
