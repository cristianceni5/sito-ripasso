0. Algoritmi: Definizione e Proprietà

Un algoritmo è una successione finita di istruzioni, assegnate in modo non ambiguo, la cui esecuzione consente di passare da una situazione iniziale (DATI) a una situazione finale (RISULTATI) in un tempo finito.

Requisiti fondamentali

  • Generalità: adatto a risolvere un'intera classe di problemi, non un caso specifico.
  • Ottimalità: minimizzare il tempo, il numero di operazioni e massimizzare la stabilità.
  • Finitezza: deve terminare in un numero finito di passi.
  • Non ambiguità: ogni istruzione deve essere interpretabile in un solo modo.

Proprietà di un algoritmo numerico

  • Costo: risorse richieste — tempo di calcolo (numero di operazioni floating point, flops) e memoria.
  • Stabilità: sensibilità agli errori introdotti durante la computazione. Un algoritmo instabile amplifica gli errori di arrotondamento.

Algoritmo di Horner (Valutazione di Polinomio)

Dato il polinomio Pn(x) = a0 + a1x + a2x² + ... + anxⁿ, l'algoritmo banale richiede n addizioni + n(n+1)/2 moltiplicazioni. La Regola di Horner riscrive il polinomio in forma annidata: 

// P(x) = a0 + x*(a1 + x*(a2 + ... + x*an))
// → solo n addizioni + n moltiplicazioni (ottimale!)
Algoritmo di Horner:
  1. Leggi n, a[0..n], α
  2. s = a[n]
  3. Per i = n-1, n-2, ..., 0:
     3.1  s = s * α + a[i]
  4. Restituisci s   // s = P(α)

Costo: n moltiplicazioni + n addizioni (vs n² moltiplicazioni del metodo diretto).

Criteri di Arresto (metodi iterativi)

Un metodo iterativo genera una successione xk → x*. Non potendo eseguire infinite iterazioni, si fermano quando soddisfano un criterio:

  • Residuo: ‖b - Axk‖ < tolleranza
  • Incremento: ‖xk+1 - xk‖ < tolleranza
  • Numero massimo di iterazioni: k > k_max (sicurezza contro divergenza)

1. Aritmetica Finita e Rappresentazione in Macchina

Sistemi di Numerazione

I computer lavorano in base β (solitamente β=2, binario). Un numero x in notazione posizionale in base β con t cifre di mantissa e p cifre di caratteristica si scrive:

x = ±(d₁β⁻¹ + d₂β⁻² + ... + dtβ⁻ᵗ) × βᵉ

dove:  dᵢ ∈ {0, 1, ..., β-1}  (cifre)
       d₁ ≠ 0                  (forma normalizzata)
       e   ∈ [L, U]            (esponente / caratteristica)

Numeri Interi in Macchina

Con n bit disponibili e 1 bit di segno, si rappresentano interi da -(2n-1) a +(2n-1)-1. Se il numero supera questo range si verifica OVERFLOW.

Virgola Mobile Normalizzata (Floating Point)

Un numero reale x è rappresentato come fl(x) — la sua approssimazione floating point. I parametri della macchina sono: base β, cifre mantissa t, esponente [L, U].

ComponenteDescrizioneEsempio (IEEE 754 double)
Segno1 bit: 0 positivo, 1 negativo1 bit
Mantissat cifre binarie — determina la precisione52 bit
Caratteristica (esponente)Determina l'ordine di grandezza11 bit, bias 1023

Metodi di Approssimazione

MetodoFormulaErrore assoluto massimo
Troncamentofl(x) = tronca le cifre eccedenti|x - fl(x)| ≤ β1-t
Arrotondamentofl(x) = arrotonda all'ultima cifra|x - fl(x)| ≤ ½ β1-t

Precisione di Macchina (εmach)

La precisione di macchina εmach è il più piccolo numero positivo tale che fl(1 + εmach) > 1 in aritmetica float. Misura la precisione relativa massima:

ε_mach = β^(1-t)      (troncamento)
ε_mach = ½ β^(1-t)   (arrotondamento)

IEEE 754 double: ε_mach ≈ 2.22 × 10⁻¹⁶
IEEE 754 single: ε_mach ≈ 1.19 × 10⁻⁷

Overflow e Underflow

  • Overflow: |x| > xmax = (1 - β-t) × βU+1 → il numero è troppo grande da rappresentare.
  • Underflow: 0 < |x| < xmin = βL-1 → il numero è troppo piccolo e viene azzerato.

Errori di Rappresentazione

Dato x e la sua approssimazione fl(x):

Errore Assoluto:  |x - fl(x)|
Errore Relativo:  |x - fl(x)| / |x|    (x ≠ 0)

L'errore relativo NON dipende dall'ordine di grandezza
→ misura la precisione della mantissa
→ sempre ≤ ε_mach

Cancellazione Numerica

Sottraendo due numeri molto vicini si perdono cifre significative. Esempio: x = 1.000001, y = 1.000000 → x - y = 0.000001 (solo 1 cifra significativa rimasta).
Rimedio: riformulare algebraicamente le espressioni per evitare sottrazioni di valori vicini.

2. Sistemi Lineari: Ax = b

Dato A ∈ ℝn×n e b ∈ ℝn, trovare x ∈ ℝn tale che Ax = b. Se det(A) ≠ 0 esiste unica soluzione x = A-1b (ma in pratica non si calcola mai A-1!).

Eliminazione di Gauss (Metodo Diretto)

Trasforma il sistema Ax=b in un sistema equivalente triangolare superiore Ux=c mediante operazioni elementari sulle righe (fattore di annullamento mij = aij/aii).

Per k = 1 a n-1:        (colonna pivot)
  Per i = k+1 a n:      (righe sotto)
    m_ik = a_ik / a_kk  (moltiplicatore)
    Per j = k a n:
      a_ij = a_ij - m_ik * a_kj
    b_i = b_i - m_ik * b_k

Costo: O(n³/3) moltiplicazioni/divisioni + O(n²/2) per la sostituzione all'indietro

Fattorizzazione LU

L'eliminazione di Gauss equivale a fattorizzare A = L·U dove L è triangolare inferiore (con 1 sulla diagonale, moltiplicatori mij sotto) e U è triangolare superiore (risultato dell'eliminazione).

A = L · U
Ly = b       (sostituzione in avanti,  costo O(n²))
Ux = y       (sostituzione all'indietro, costo O(n²))

Vantaggio: con più vettori b diversi, si fattorizza A una volta sola.

Pivoting Parziale

A ogni passo k si sceglie come pivot il massimo elemento in valore assoluto nella colonna k (sotto la diagonale), e si scambia quella riga con la riga k. Questo previene la divisione per zero e riduce l'amplificazione numerica: PA = LU con P matrice di permutazione.

Sistemi Triangolari: Sostituzioni

// Sostituzione in AVANTI (Ly = b, L triangolare inferiore)
y[1] = b[1] / L[1,1]
Per i = 2 a n:
  y[i] = (b[i] - somma(L[i,j]*y[j], j=1..i-1)) / L[i,i]

// Sostituzione ALL'INDIETRO (Ux = y, U triangolare superiore)
x[n] = y[n] / U[n,n]
Per i = n-1 a 1 (decrescente):
  x[i] = (y[i] - somma(U[i,j]*x[j], j=i+1..n)) / U[i,i]

3. Norme Vettoriali e Matriciali, Condizionamento

Norme Vettoriali

Una norma ‖·‖: ℝn → ℝ deve soddisfare: (1) ‖x‖ ≥ 0 e ‖x‖=0 ⟺ x=0; (2) ‖αx‖ = |α| ‖x‖; (3) ‖x+y‖ ≤ ‖x‖ + ‖y‖ (disuguaglianza triangolare).

NormaFormulaGeometria
Norma 1‖x‖₁ = Σ|xᵢ|Distanza di Manhattan
Norma 2 (euclidea)‖x‖₂ = √(Σxᵢ²) = √(xᵀx)Distanza euclidea classica
Norma ∞‖x‖∞ = max|xᵢ|Massimo in valore assoluto

Norme Matriciali (Indotte)

La norma matriciale indotta dalla norma vettoriale ‖·‖ è: ‖A‖ = supx≠0 ‖Ax‖/‖x‖. Proprietà aggiuntiva: ‖AB‖ ≤ ‖A‖·‖B‖ (submoltiplicatività).

Numero di Condizionamento K(A)

Misura quanto una perturbazione sui dati (b o A) si amplifica nella soluzione x. È definito come:

K(A) = ‖A‖ · ‖A⁻¹‖     (rispetto a una norma scelta)

Proprietà:
  K(A) ≥ 1  sempre
  K(I) = 1  (matrice identità = perfettamente condizionata)
  K(A) = ∞  se A è singolare

Stima dell'errore relativo (perturbazione solo su b):
  ‖δx‖/‖x‖  ≤  K(A) · ‖δb‖/‖b‖

Con perturbazione anche su A:
  ‖δx‖/‖x‖  ≤  K(A) · (‖δb‖/‖b‖ + ‖δA‖/‖A‖)

Se K(A) è grande (es. 106), la matrice è malcondizionata: piccole perturbazioni nei dati causano grandi errori nella soluzione. Attenzione: det(A) grande non implica buon condizionamento! (vedi Matrice di Hilbert)

4. Approssimazione di Dati e Funzioni

Interpolazione Polinomiale

Dati n+1 punti (x0,y0), ..., (xn,yn) con nodi distinti, esiste un unico polinomio Pn(x) di grado ≤ n tale che Pn(xᵢ) = yᵢ per tutti i nodi. (Teorema di esistenza e unicità dell'interpolante polinomiale).

Formula di Lagrange

Il polinomio interpolante si scrive come combinazione lineare dei polinomi base di Lagrange Lk(x):

Pₙ(x) = Σₖ₌₀ⁿ  yₖ · Lₖ(x)

dove:  Lₖ(x) = Π(x - xⱼ)/(xₖ - xⱼ)   j=0..n, j≠k

Proprietà: Lₖ(xⱼ) = δₖⱼ  (1 se k=j, 0 altrimenti)

Problema di Runge: con nodi equidistanti e grado elevato, il polinomio oscilla violentemente agli estremi. Rimedio: usare nodi di Chebyshev.

Formula di Newton (Differenze Divise)

Forma più efficiente computazionalmente. Il polinomio si costruisce aggiungendo un nodo alla volta:

Pₙ(x) = f[x₀] + f[x₀,x₁](x-x₀) + f[x₀,x₁,x₂](x-x₀)(x-x₁) + ...

Differenze divise:
  f[xₖ] = yₖ                                           (ordine 0)
  f[xₖ,xₖ₊₁] = (f[xₖ₊₁] - f[xₖ]) / (xₖ₊₁ - xₖ)     (ordine 1)
  f[xₖ,...,xₖ₊ⱼ] = (f[xₖ₊₁,...,xₖ₊ⱼ] - f[xₖ,...,xₖ₊ⱼ₋₁]) / (xₖ₊ⱼ - xₖ)

Costo: O(n²) per costruire la tabella delle differenze divise, O(n) per valutare con Horner.

Errore di Interpolazione

f(x) - Pₙ(x) = f[x₀,...,xₙ,x] · ω_{n+1}(x)

dove ω_{n+1}(x) = (x-x₀)(x-x₁)...(x-xₙ)

Se f ∈ C^{n+1}[a,b]:
  |f(x) - Pₙ(x)| ≤ max|f^{(n+1)}(ξ)| / (n+1)! · |ω_{n+1}(x)|

Minimi Quadrati (Least Squares)

Quando i dati sono rumorosi (molti punti, imprecisi), non ha senso interpolare tutti. Si cerca il polinomio di grado m << n che minimizza la somma degli scarti quadratici:

min Σᵢ (Pₘ(xᵢ) - yᵢ)²

→ Sistema delle equazioni normali: AᵀA c = Aᵀy
  dove A è la matrice di Vandermonde (m colonne)
  c sono i coefficienti del polinomio da trovare

5. Integrazione Numerica (Quadratura)

Approssima ∫ab f(x)dx con una somma pesata dei valori di f in certi nodi:

I(f) = ∫ₐᵇ f(x)dx  ≈  Iₙ(f) = Σᵢ₌₀ⁿ wᵢ f(xᵢ)

dove xᵢ sono i nodi e wᵢ i pesi di quadratura.
MetodoNodiPesoOrdine di erroreNote
Punto Medioc = (a+b)/2w = b-aO(h²)1 nodo, polinomi grado ≤1 esatti
Trapezia, bw = h/2 eachO(h²)2 nodi, semplice
Simpsona, (a+b)/2, bh/6, 4h/6, h/6O(h⁴)3 nodi, esatto per grado ≤3

Formule Composite

Si suddivide [a,b] in N sottointervalli di ampiezza h = (b-a)/N e si applica la formula semplice su ciascun sotto-intervallo:

Trapezi Composita:
  Iₙ(f) = h/2 · (f(x₀) + 2f(x₁) + 2f(x₂) + ... + 2f(xₙ₋₁) + f(xₙ))
  Errore: -(b-a)/12 · h² · f''(ξ)   → O(h²)

Simpson Composita (N pari):
  Iₙ(f) = h/3 · (f(x₀) + 4f(x₁) + 2f(x₂) + 4f(x₃) + ... + f(xₙ))
  Errore: -(b-a)/180 · h⁴ · f''''(ξ)  → O(h⁴)

6. Equazioni Non Lineari: f(x) = 0

Trovare la radice x* tale che f(x*) = 0. In generale non esistono formule chiuse → metodi iterativi.

Metodo di Bisezione

Se f continua su [a,b] e f(a)·f(b) < 0 → esiste radice in (a,b) (teorema degli zeri). Si dimezza l'intervallo ad ogni passo.

Algoritmo Bisezione:
  Dati a, b  con f(a)·f(b) < 0, tolleranza tol
  Mentre (b-a)/2 > tol:
    c = (a+b)/2
    Se f(c) = 0: STOP (radice esatta)
    Se f(a)·f(c) < 0: b = c
    Altrimenti:        a = c
  Restituisci c = (a+b)/2

Errore dopo k passi: |cₖ - x*| ≤ (b-a)/2^(k+1)
→ Convergenza LINEARE, lenta ma garantita.

Metodo di Newton-Raphson

Usa la derivata f'(x) per trovare la radice molto più rapidamente (convergenza quadratica).

x₀ dato (stima iniziale vicina a x*)
xₖ₊₁ = xₖ - f(xₖ) / f'(xₖ)

Interpretazione geometrica: xₖ₊₁ è il punto dove la retta tangente
al grafico di f in xₖ interseca l'asse x.

Convergenza QUADRATICA: |eₖ₊₁| ≈ C · |eₖ|²  (se f'(x*) ≠ 0)
→ il numero di cifre corrette raddoppia ad ogni iterazione!

Limiti: richiede f'(x), può divergere se x₀ è lontano, f'(x*)=0 rallenta.

Metodo della Secante

Come Newton ma approssima f'(x) con il rapporto incrementale tra due iterati precedenti (non serve calcolare la derivata).

x₀, x₁ dati
xₖ₊₁ = xₖ - f(xₖ) · (xₖ - xₖ₋₁) / (f(xₖ) - f(xₖ₋₁))

Convergenza superlineare (ordine ≈ 1.618, sezione aurea)
→ più lenta di Newton ma non richiede la derivata.

Metodo del Punto Fisso

Si riscrive f(x)=0 nella forma x = g(x). Partendo da x₀ si itera xk+1 = g(xk).

Teorema di convergenza:
  Se g: [a,b] → [a,b] e |g'(x)| ≤ L < 1 per ogni x in [a,b]
  → il punto fisso x* = g(x*) esiste unico e la successione converge.

L è il "fattore di contrazione": più L è piccolo, più rapida la convergenza.
Convergenza LINEARE con fattore L: |eₖ₊₁| ≤ L · |eₖ|

Esempio: trovare radice di x² - 2 = 0
  g(x) = (x + 2/x) / 2  (metodo di Erone per √2)
