Corso C++ (Programmazione ad Oggetti)

Sintesi completa degli appunti del corso MICC: Introduzione al C++, Classi, Oggetti ed Ereditarietà

1. Introduzione al C++ e OOP

Il C++ è un'estensione del linguaggio C creata da Bjarne Stroustrup, che introduce il paradigma della Programmazione Orientata agli Oggetti (OOP). Mantiene la retrocompatibilità (quasi totale) col C ma offre strumenti più potenti per gestire sistemi software complessi.

Punti Chiave (Rispetto al C)

  • Classi e Oggetti: Le struct del C diventano classi (aggregati di dati e metodi che operano su di essi).
  • Incapsulamento: Nascondere i dettagli implementativi (Information Hiding).
  • Ereditarietà e Polimorfismo: Creazione di gerarchie di tipi e comportamenti dinamici a runtime.
  • Libreria Standard: Introduzione della STL (Standard Template Library), di std::string e dei flussi I/O (std::cout, std::cin).
// Esempio base in C++ (HelloWorld)
#include <iostream>

int main() {
    std::cout << "Hello, World!" << std::endl;
    return 0;
}

2. Classi e Oggetti

Una Classe è un "progetto" o un "tipo" definito dall'utente. Un Oggetto è un'istanza (concreta e in memoria) di una classe. Ogni classe definisce attributi (variabili membro) e metodi (funzioni membro).

Access Specifiers (Visibilità)

  • public: Accessibile da qualsiasi punto del codice. Di solito usato per l'interfaccia (i metodi).
  • private: Accessibile solo dall'interno della classe stessa. (Visibilità di default in C++). I dati dovrebbero sempre essere privati.
  • protected: Come il private, ma accessibile anche dalle classi derivate.

Esempio: File Header (.h) vs Implementazione (.cpp)

In un progetto reale (es. le esercitazioni E1_class_exercise), si separa la dichiarazione della classe (nel file header) dalla sua definizione (nel file cpp).

// --- GameCharacter.h ---
class GameCharacter {
public:
    // Costruttore (inizializza l'oggetto)
    GameCharacter(int hp = 10, int a = 10);
    
    // Metodi pubblici (Interfaccia)
    void move(int distance);
    void move(int x, int y); // Overloading del metodo
    int getHP();
    void receiveDamage(int points);

private:
    // Dati privati (Incapsulamento)
    int HP;
    int armor;
    int posX, posY;
};

3. Ereditarietà e Polimorfismo (Virtual/Override)

L'Ereditarietà permette a una nuova classe (Derived) di ereditare proprietà e metodi da una classe esistente (Base). È una relazione di tipo "Is-A" (È un).

Polimorfismo e Funzioni Virtuali

Il polimorfismo fa sì che la stessa chiamata a funzione si comporti diversamente a seconda del tipo o dell'oggetto reale a runtime. Per farlo in C++, il metodo nella classe base deve essere marcato come virtual.

// --- Wizard.h (Deriva da GameCharacter) ---
#include "GameCharacter.h"

class Wizard : public GameCharacter {
public:
    Wizard(std::string n, int m) : GameCharacter(), name(n), mana(m) {}

    // Override: Sostituisce il comportamento del metodo base
    char getCharacterSymbol() const override {
        return 'W';
    }

    virtual void doMagic(); // Metodo nuovo, marcato virtual per ulteriori estensioni

protected:
    // Accessibile alle eventuali specializzazioni di Wizard
    int mana;
    std::string name;
};

Overloading vs Overriding:
Overloading: Stesso nome di funzione, ma parametri (signature) diversi, risolto a compile-time (es. i costruttori o i metodi move()).
Overriding: Stesso nome e stessi parametri di una funzione virtuale della classe base, ma in una classe derivata. Risolto a runtime.
